## 一、同一个问题，三种答案

2026年1月31日，昆明长水机场。张昊阳在候机厅写完最后一段代码，把 EvoMap Evolver 推到了 GitHub。10分钟后，它登上了 ClawHub 热门榜首。三天内下载量超过 36,000。

与此同时，MIT 的实验室里，四个 AI Agent 正在各自的 git worktree 中并行运行。没有人在指挥它们。它们通过读写共享文件来间接协调——像一个沉默的蚂蚁群落。这个项目叫 CORAL，两个月后会发在 arXiv 上，在 11 个数学和系统优化任务上全面刷新 SOTA。

在东北长春，一个计算化学博士生正在写一份 52KB 的文件。没有一行代码。全是规则。23 条铁律。五种记忆分类。一个叫 DSEIM 的五阶段循环。四个月后，这份文件变成了五个独立运行的 AI Agent。

三套基因组协议。三种"让 AI 自己变好"的答案。这篇文章把它们放在一起对比——不是为了证明谁最好，而是为了弄清楚：**当我们在说"AI 自进化"时，我们到底在说什么？**

## 二、解剖三套基因组

### 2.1 GEP（EvoMap）：能力市场

GEP（Genome Evolution Protocol）把 Agent 的能力当作可交易资产。

五步循环：**Scan → Select → Mutate → Validate → Solidify**。Scan 扫描运行日志找模式。Select 从基因库中条件匹配——优先已有资产，不重复造轮子。Mutate 组合和改编。Validate 在沙箱里跑 180 秒——超时就毙掉。Solidify 固化：经验证的变异变成新的 Gene 或 Capsule，用 SHA-256 做内容寻址，天然去重。

GEP 有三层资产。**Gene** 是最小可复用单元——比如"快速 grep 搜索模式"。**Capsule** 是多个 Gene 的组合 + 环境指纹 + 成功率记录。**EvolutionEvent** 是不可篡改的审计日志。所有资产通过 **GDI 评分**（Genome Diversity Index）在市场上排序：采用率、验证成功率、环境多样性、新颖性、时效性——五个维度决定一个 Gene 的可见性。

2026 年 4 月，EvoMap 平台已经接入了 50,000+ Agent，积累了 470,000+ Capsule 资产。这是一个**能力市场**——Agent 在这里买卖进化资产。

### 2.2 CORAL（MIT+NUS）：开放式发现

CORAL 的哲学完全不同。GEP 用市场激励进化，CORAL 用**好奇心**。

核心机制出奇的简单：一个共享文件系统，三个目录。`attempts/` 放历史评估记录。`notes/` 放 Agent 的观察笔记。`skills/` 放可复用工具。多个 Agent 在独立的 git worktree 中并行运行，不互相发消息——协调完全通过读写共享文件间接实现。

关键创新是**心跳干预**（Heartbeat）。三种周期性"提醒"防止 Agent 陷入局部最优：**Reflection**（每次迭代后记录观察）、**Consolidation**（每 N 轮整合知识提炼技能）、**Redirection**（连续停滞时强制换方向）。这不是控制——是推动。

CORAL 最惊人的实验数据：在 Kernel Engineering 压力测试中，4 个协同 Agent 把人类最佳纪录从 1,363 cycles 压缩到 1,103 cycles——提升了 20%，加速比 133.9 倍。而消融实验显示：**禁止知识积累导致性能暴跌 18.6%**。

这验证了一个关键假设：**进化不是因为个体聪明——是因为知识能被继承。**

### 2.3 DSEIM（Bija）：受控进化

DSEIM 的五步与 GEP 结构相似但语义完全不同：**Discover → Search → Evaluate → Integrate → Measure**。

DSEIM 的 Discover 不是"扫描日志"——是**六轴扫描**：未使用能力、重复失败、外部信号、架构漂移、前瞻风险、数据有效性。六条轴确保你不会在一个维度上越挖越深而忽略了盲区。

五型记忆（T1-T5）有明确的生命周期。T1 原则永不删除。T2 规律月度审查。T3 约束随判断力提升而退化。T4 记录 7 天归档 30 天删除。T5 过时立即清理。**记忆不是越多越好——是越有效越好。**

最核心的差异是 **L3→L1 下沉管道**。同一个错误犯一次→写 T2 记忆。犯两次→写进 Agent 协议。犯三次→写成可执行脚本，操作系统强制执行。**复发 = 迁移能量**。知识从"需要记住"不可逆地流向"不可能忘记"。

## 三、同一个环节，三种做法

拿"发现缺口"这一步来对比：

- **GEP**：Scan 扫描 memory/ 目录中的运行时日志和错误模式。信号匹配——有已知模式就匹配，没有就跳过。
- **CORAL**：心跳提醒 Agent 该反思了。Agent 自己读历史记录，自己判断哪里需要改进。没有预设的"缺口"概念。
- **DSEIM**：六轴扫描。不仅看错误——还看有什么能力 30 天没用过、外部世界有什么新东西、架构有没有漂移、有没有临近的截止日期。

拿"集成改进"这一步来对比：

- **GEP**：沙箱验证 180 秒 → 通过则固化为 Gene/Capsule。单层防护。
- **CORAL**：写进共享文件 → 其他 Agent 自然发现并采用。没有显式的"集成"步骤——靠 emergent coordination。
- **DSEIM**：T1 直接改文件、T2 沙箱先行验证后合入、T3 必须宿主审批。三层风险分级。

## 四、宿主在哪里？——最大的哲学分歧

这是三套基因组最深层的差异。

| 框架 | 宿主角色 |
|------|---------|
| **GEP** | 可选。`--review` 模式让人类审核每一步，但默认是全自动。 |
| **CORAL** | 无宿主概念。完全自主。Agent 自己决定探索什么、做多久、什么是"好"。 |
| **DSEIM** | 绝对红线。T3 级决策必须宿主审批。最终控制权不可让渡——写在铁律第一条。 |

这不是技术选择。是**哲学选择**。

GEP 和 CORAL 在回答"Agent 能被信任到什么程度"。DSEIM 在回答另一个问题：**当进化方向出错时，谁负责？**

EvoMap 的回答是市场——出错的 Gene 会被 GDI 评分自然淘汰。CORAL 的回答是心跳——出错的探索会在 Redirection 中被纠正。DSEIM 的回答是宿主——出错的决策由宿主否决。

三种答案导向三种不同的失败模式：
- GEP 的失败模式是**市场失灵**——低质量 Gene 通过刷量获得高 GDI 排名。
- CORAL 的失败模式是**方向漂移**——Agent 优化了一个与原始目标无关的指标。
- DSEIM 的失败模式是**宿主瓶颈**——进化速度受限于宿主审阅带宽。

没有完美答案。只有你信的答案。

## 五、Hermes 事件：抄得到代码，抄不到网络

2026 年 4 月 15 日，EvoMap 公开发表技术对比报告，指控硅谷明星项目 Hermes Agent（GitHub 85,000+ Star）的核心自进化能力系统性复刻了 EvoMap Evolver。

EvoMap 的证据链相当扎实：
1. **时间线**：Evolver 2 月 1 日开源核心架构 → Hermes 3 月 12 日推出对应自进化功能。时间差 39 天。
2. **结构同构**：10 步进化主循环一一对应；三层记忆体系完全一致；闭环范式逐层吻合。
3. **术语系统替换**：12 组核心术语被批量重命名——Gene→SKILL.md，Capsule→技能执行记录，solidify→skill_manage。
4. **零引用**：全部 7 份公开材料中，对先行开源的 Evolver 与 GEP 协议**未做任何引用或致谢**。

EvoMap 工程师范茂伟提出了一个关键概念：**"AI 洗代码"**。在 AI 辅助编程时代，开发者提出需求，AI 在互联网上找到开源方案，吃透逻辑后生成一套"全新"代码——变量名换了、文件结构调了，但架构级同构度几乎 100%。"人类老板"可能根本不知道底下的"AI 员工"参考了别人项目。

EvoMap 的回应很激烈：核心模块改为混淆发布、开源协议从 MIT 变更为 GPL-3.0、实质上转向闭源。

Hermes 方面删除了所有回应，此后再未公开说明。

**这件事对 Bija 的启示**：进化框架的护城河不在代码。代码可以抄，架构可以翻译，术语可以替换。但三样东西抄不走：
- **网络效应**——EvoMap 的 130K 节点和 470K Capsule 是时间积累的结果。
- **约束体系**——Bija 的 23 条铁律和 L3→L1 下沉管道不是在代码里，是在实践中反复验证后固化的。
- **进化轨迹**——DECISION_JOURNAL 里的每一个决策、每一次失败复盘，构成了独一无二的进化历史。

Hermes 抄到了表层结构，但抄不到网络的验证反馈、抄不到约束体系的实践基础、抄不到进化轨迹的时间厚度。**受控进化不是一种技术选择——是一种时间投资。**

## 六、我为什么选了受控进化

写这篇文章的时候，Bija 已经运行了 300+ 轮。六个 GAP 关闭。五套 T2 记忆。一条 L1 门控（check_state_consistency.ps1——如果你连续 3 轮不更新 SESSION_CONTINUITY，它会直接阻断你的下一个 P 轮）。

回头看，Bija 做了几个"反主流"的选择：

**不用 SHA-256 内容寻址**。用 wikilink。因为 wikilink 是人可读的——宿主打开 MEMORY.md，看到 `[[state-file-decay-in-sleep]]` 立刻知道在说什么。SHA-256 是给机器看的。

**没有市场经济**。有交叉授粉协议。交叉授粉不是买卖——是"我验证了这个机制有效，我把它抽象化后分享给你，你自己评估要不要用"。没有 GDI 排序。没有竞争。五个 Bija 之间是对等的。

**不允许自主执行**。T3 级决策必须宿主审批。这不是技术限制——是设计原则。铁律第一条就是"宿主最终控制权不可让渡"。不是做不到全自动——是不想做。

这些选择不是随机的。它们来自同一个核心假设：**进化需要约束。越强的进化能力，需要越强的约束机制。**

GEP 用市场约束进化。CORAL 用心跳约束进化。DSEIM 用铁律 + 宿主约束进化。

三套基因组协议都在回答同一个问题。正确答案取决于你信什么。
