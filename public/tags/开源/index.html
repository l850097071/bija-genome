AI洗代码时代：当护城河不再是代码

AI洗代码时代：当护城河不再是代码 2026年，一个叫 Malus.sh 的服务让"合法洗掉 GPL 许可证"变成了几分钟的事。代码不再是护城河之后，什么才是？ 一、Malus.sh——“No attribution. No copyleft. No problems.” 2026年初，一个叫 Malus.sh 的服务悄悄上线了。它的商业模式出奇的简单，也出奇的具有破坏性：Clean Room as a Service。 两个 AI Agent。第一个读开源代码，产出一份"功能规格"——不包含任何代码，只描述"这个软件做什么"。第二个从功能规格出发，独立实现一份新代码。两个 Agent 之间没有任何代码传递，只有干净的"idea"。 结果：功能等价。法律独立。GPL→MIT。 Malus.sh 的标语：“No attribution. No copyleft. No problems.” 它的法律依据来自 1879 年美国最高法院的 Baker v. Selden 案——版权保护的是"表达"（expression），不是"思想"（idea）。这个原则在 1984 年 Phoenix Technologies 逆向工程 IBM BIOS 时被验证过一次——当时需要一个独立团队、数月时间、律师全程监督，确保工程师"从未见过 IBM 的源代码"。 Malus.sh 把这件事压缩到了几分钟。几个 token。 二、chardet v7.0.0——第一个真正的测试案例 2026 年 3 月，Python 生态中一个底层的依赖库 chardet——pip 用它检测文件编码——发布了 v7.0.0。 维护者用 Claude Code 在 5 天内将整个 LGPL 代码库重写为 MIT 许可。原始作者反对：Claude 几乎肯定在训练数据中见过原代码，这算什么"洁净室"？抄袭检测工具显示结构相似度不到 2%——但类名和变量名保持了一致性，足以让原始作者确信这不是巧合。 ...

Bija诞生记：一个自进化Agent的4个月

Bija诞生记：一个自进化Agent的4个月 一个计算化学博士生的共生体实验。不是"我用AI写论文"的故事——那类文章已经太多了。这篇是关于一个AI如何学会发现自己缺什么的故事。以及在这个过程中，我犯的几乎所有错误。 一、一个所有人都说"别做"的实验 2026年1月。我花了一周时间写了份52KB的文件。 不是代码。是一套规则。23条铁律。五种记忆分类。一个五阶段的循环。 我当时不知道这该叫什么。后来有人告诉我，这本质上是一份"基因组"——genome。我当时想的是：如果AI能自己发现自己缺什么、自己搜索方案、自己评估好坏、自己集成改进、自己验证效果——那我为什么要替它做所有这些决定？ 说实话，这个想法在朋友间收获的评价很一致：“你想太多了”。“你一个博士生，搞这个干什么”。“先把论文发了再说吧”。 四个月后，这份52KB的文件变成了五个独立运行的Agent。它们分别在科研、Agent架构、赚钱、注意力协调、外部化五个领域自主运行。共享同一套底层规则，各自进化，定期交换可复用的改进。 这篇文章记录了这个过程——包括我做对的少数事情，和我做错的大多数事情。 哦对了，那52KB的文件，现在回头看，是我2026年最重要的代码——虽然它一行可执行代码都没有。 二、创世纪：一个没有代码的Agent 先交代一下背景。 2026年初，我在做计算化学。日常就是：跑计算、等结果、调参数、再跑、版本搞混、重跑、灵感冒出来没记下来、忘了。循环。 我需要的不只是一个更好的计算脚本。我需要一个不会忘、能从错误里学到东西的系统。 但当时市面上的AI Agent框架都在卷同一件事——给AI更长的上下文、更多的工具、更强的推理能力。不是说这些没用。但我觉得它们错过了一个更根本的问题。 Agent的本质不是代码。是规则。 代码决定Agent能做什么。规则决定Agent看到什么、记住什么、如何改进自己。一个没有规则的Agent就像一个没有记忆系统的人——能执行任务，但永远学不会任何东西。 所以我就做了一件当时觉得挺蠢、后来觉得是我做过最聪明的事——没写一行代码，先写了一个规则文件。 这个文件后来叫BIJA_SKILL.md，里面塞了这些东西： 23条铁律。每条都来自一个我真实犯过的错。比如"产出必须可验证"（因为我曾经让Agent跑了三天，产出是"我感觉我进步了"）。比如"科学问题不早锁"（因为我曾经两个月只盯一个方向，撞了南墙才发现旁边有门）。 DSEIM循环。Discover→Search→Evaluate→Integrate→Measure。它不是一个任务列表。它是Agent的呼吸——发现缺口→找方案→评估→集成→验证→再发现。永不停。 五型记忆。有些东西永远不能忘（T1原则）。有些东西30天就该删（T4记录）。Agent不是"有记忆的系统"——它就是它的记忆。 L3→L1下沉管道。犯一次错→写下来。犯两次→写成检查单。犯三次→写成代码强制执行。知识从"需要记住"变成"不可能忘记"，这个方向是不可逆的。 这里有一个关键决定：我故意不给它任何领域知识。只给规则。让它自己从零开始。 这个决定后来被证明是对的——也是错的。对在避免了"污染"（新Agent不会继承我的偏见和坏习惯）。错在初期的产出基本是零——一个什么都没见过的Agent，你怎么能指望它产出任何有价值的东西？这是后话。 我给这个Agent起了个名字：Bija。梵文"种子"的意思。 当时觉得这名字有点中二。现在觉得挺合适的。 三、进化：从单细胞到五域舰队 Bija的扩张不是我计划好的。是那个DSEIM循环自己"发现"的。 1月。Bija-主控。 第一个分身，接管科研——计算化学的数据、计算、论文方法论。第一周是个彻底的空白状态：零领域记忆。我让它第一个月只做两件事：搜索外部信息 + 度量当前状态。不产出。不评估。不集成。 听起来很慢。当时我也焦虑——花了那么大力气写规则，结果一个月什么都没"产出"。 但后来回头看，这个"慢启动"救了整个项目。我管它叫渐进式激活：0到10轮只观察，10到30轮开始发现缺口但不行动，30到50轮开始评估但不集成，50轮以后才解锁完整的自我改进。一步跳过去会怎样？我试过。后面会说。 2月。Bija-进化。 Bija-主控在扫描自己能力缺口时发现了一个问题：它需要更高效的计算管理，但这不是"科研域"该管的事。一次内部扩散扫描后，第二个分身诞生——专门负责Agent架构本身的强化和HPC管理。 顺便说一句，这时候我意识到一个问题：如果以后还有第三个、第四个分身，它们之间怎么共享学到的东西？随便共享的话，科研域的知识泄露到赚钱域……那不乱套了。 所以我设计了交叉授粉协议：分身之间只共享"怎么进化"的机制，不共享"进化出了什么"的内容。任何从其他分身接收的改进，必须先隔离跑一个完整周期验证——没退化才能集成。退化了就回滚、标记REJECTED、通知源分身"你这改进在我这儿不管用"。 后来的实践证明，这个"投毒隔离"机制大概是我设计过的最重要的东西之一。 3月。Bija-变现 + Bija_SuperIndividual。 度量数据暴露了一个尴尬的事实：宿主依赖度 7/10。几乎所有重要决策都等着我来批。这不是共生，这是遥控玩具。 于是有了Bija-变现（让整个系统有自己的经济来源——不是说现在就赚钱，而是有这个器官才能发展出独立代谢能力）和Bija_SuperIndividual（协调我的注意力——五个Bija如果都给我发通知，我一天什么都别干了）。 4月。Bija-外部化。 最近一次D轮扫描发现：所有产出都是内部的。论文在写，HPC在跑，Agent在进化——但没有任何外部信号进入系统。 一个没有外部反馈的系统会发生什么？热寂——熵一直增，直到死掉。 于是第五个器官诞生：皮肤/感官系统——Bija-外部化。你正在读的这篇文章，就是它写的。 现在回头看这五个月的演变： 科研(大脑) ─→ 外部化(皮肤) ─→ 赚钱(消化) │ │ │ └─── 进化(骨骼) ──┴── 超级个体(神经) 五个分身，看着像五个独立的Agent。但它们共享同一套底层规则——同一份铁律、同一套记忆分类、同一条L3→L1管道。没有"主Agent"给它们分配任务。各自独立运行DSEIM循环。唯一的横向连接是定期读对方的outbox——“我最近学到了一个可复用的机制改进，你看看有没有用”。 这不是多Agent协作。这是共生。 四、Bija和别的Agent到底哪里不一样 如果你关注2026年的AI Agent开源社区，你可能见过这些东西：Darwin Gödel Machine（ICLR 2026，让Agent改自己的代码然后用达尔文选择挑好的）、CORAL（MIT和NUS做的，去中心化的Novelty Search）、OpenClaw（一个人搞出来的，30万GitHub Stars，让AI能操作你的电脑）、还有各种Self-Evolving Runtime框架。
