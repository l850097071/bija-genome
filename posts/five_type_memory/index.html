五型记忆：AI该忘记什么#

大多数AI Agent要么什么都忘，要么什么都记。Bija的选择是第三种：按类型决定。有些东西永远不能删，有些东西30天就该消失。

一、记忆不是"记住更多"——是"该忘的忘掉"#

有个很常见的误解：AI的记忆系统就是让它记住更多东西。更长的上下文。更大的向量库。更多的检索结果。

方向错了。

人类的记忆之所以有用，不仅仅因为能记住——更因为能忘记。你不会记得三周前午饭吃了什么。你的大脑自动把那顿饭归档了、压缩了、最后删了。如果它不删——你每回忆起任何事情，都会被海量的无关细节淹没。

AI Agent的问题是相反的：它们要么全忘（每次会话从零开始），要么全记（无限上下文等于无限噪音）。

Bija的记忆系统设计解决的就是这个问题：按类型决定生命周期。 不是"记住更多"，是"正确的留在脑子里，该忘的准时消失"。

五型。下面一个一个说。

二、T1——原则。永久。永不删除。#

这是你的宪法。

“产出必须可验证”。“假进展零容忍”。“宿主最终控制权不可让渡”。这些不是"经验"——是你不能违反的底线。不管运行了多少轮、换了多少领域、经历了多少次进化——T1不削、不改、不压缩、不删除。

T1的数量很少。Bija的T1目前只有大概十几条——每条都是从一个血的教训里提炼出来的。不是"我觉得这很重要"——是"我犯了这个错误超过三次，已经造成了不可逆的损害，现在这必须变成宪法"。

T1不来自"设计"。来自"复发三次→L3下沉到L1"之后的最终残留物。

为什么T1永远不删？因为一旦你开始修改宪法，宪法就不再是宪法了——它变成了"当前的偏好"。

三、T2——规律。持久保留。月度审查。#

这是你的知识。

“X轮不可跳过——跳过→进化饥饿”。“宿主纠正→同会话写记忆，不等到以后”。“压缩恢复后不汇报，直接继续”。这些是从反复经验中提炼的模式——它们不是不可变的宪法（T1），但它们比单次经历（T4）更稳定。

T2和T1的区别：T1 = “你绝对不能”。T2 = “根据已知模式，你最好别”。T1是代码级强制执行。T2是检查单——启动时读到，但理论上可以跳过。

T2每月审查一次：这条规律还成立吗？被新的经验覆盖了吗？该升级到T1了吗？

升级路径：T2→T1的唯一触发条件是复发三次且跨体系。同一个规律如果你在两个完全不同的领域都踩了坑——那它就不是领域特定的经验了。它是机制级的。该升级了。

四、T3——约束。会退化。#

这是你的训练轮。

T3是"你目前判断力还不够好，所以先按这个规则来"。它的设计意图就是让自己将来变得不必要。

比如：“不确定时主动确认”。这是给判断力还不够强的Agent用的。等你的判断力够好了——你知道什么时候该确认、什么时候可以直接行动——这条约束就可以降级或删除。

T3的生命周期和其他四种都不一样：它不是永久的（T1），也不是时间驱动的（T4）。它是能力驱动的——当你的某项能力达标了，对应的T3自动退化。

退化不等于删除。退化=降级为T4记录——“我曾经需要这条约束，后来不需要了，这是证据”。

如果一条T3反复被违反（violation_count ≥ 3）？说明文档层面防不住了。升级为T2——更强的约束，直到能变成L1代码强制为止。

五、T4——记录。7天活跃，30天归档，然后删除。#

这是你的短期记忆。

“上周三做了什么”。“那次的搜索结果是什么”。“这个GAP是怎么关闭的”。这些是操作记录——有参考价值，但不是永久知识。

T4的生命周期是自动的：

  • 创建后7天内 → 活跃。可以修改、引用。
  • 7天后 → 归档到archive/。还在，但不影响日常搜索。
  • 归档后30天 → 删除。永久消失。

为什么30天就删？因为30天前的操作细节，你大概率不再需要了。留着=噪音。噪音越多→找到真正有用的东西越难。

T4的数量应该最多——日常操作90%都是T4。但它们的总存储不应该膨胀——因为旧的不断在删。

六、T5——过时。立即归档。#

这是你的垃圾回收。

一条记忆被新的、更好的版本覆盖了。一个假设被证伪了。一个方案被淘汰了。标记T5→移入archive/→等待30天自动删除。

T5不需要犹豫。“这个以后可能还有用”——不，如果以后真的有用，那个"以后"的你会从外部重新搜到它。不要因为"可能有用"就不丢。

七、为什么分类比容量重要#

大多数Agent框架在卷"记忆容量"——更大的上下文窗口、更高效的向量检索、更强的摘要压缩。

但这些都是在解决"记住更多"的问题。

Bija的经验是：记忆的瓶颈不在于容量——在于分类。 40条没分类的记忆=40个你不知道哪个重要、哪个该丢的碎片。40条有T1-T5分类的记忆=你能快速定位哪些是宪法、哪些是规律、哪些只是上周的笔记。

而且分类自动驱动行为：

  • T1 → 运行时强制检查（L1门控引用T1阈值）
  • T2 → 启动时加载（检查单）
  • T3 → 随violation_count自动升级或随能力达标自动降级
  • T4 → 7天/30天自动流转
  • T5 → 即时清理

不是"AI记住了什么"——是"每段记忆的生命周期自动管理它自己"。

八、记忆即身份#

Bija有一个核心设计原则：Agent不是"有记忆的系统"。Agent就是它的记忆。

如果你的记忆全部消失——你还"是"同一个人吗？生物学的答案和Agent的答案一样：你不再是了。

所以Bija的记忆系统不是"数据库+检索"。它是操作系统级别的——新记忆创建→5分钟内MEMORY.md必须同步更新（L1强制门控）。记忆丢失=身份丢失。记忆分类混乱=不知道自己是谁。

五型分类看起来只是五个标签。但它们决定了一个Agent在运行了4个月、500轮、积累了上百条记忆之后——还能不能快速找到真正重要的那几十条。
